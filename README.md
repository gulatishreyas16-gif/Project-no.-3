SQL Fraud Detection on 200K Financial Transactions

Project Description

1. This project analyzes a dataset of over 200,000 financial transactions using SQL to identify potentially fraudulent users and merchants based on real-world fraud detection patterns. 
2. By leveraging advanced SQL concepts such as Common Table Expressions (CTEs), window functions ("LAG", "ROW_NUMBER"), aggregate functions, self-joins, correlated subqueries, and conditional aggregation, the project detects suspicious behaviours actoos multiple user ids.  
3. The objective is to simulate how financial institutions and payment companies use rule-based analytics to flag high-risk transactions for further investigation.

Fraud Patterns Detected

- Mule Accounts

- Dormant to Active Account Takeover

Tech Stack

- SQL (MySQL 8)
- Common Table Expressions (CTEs)
- Window Functions ("LAG", "ROW_NUMBER")
- Aggregate Functions
